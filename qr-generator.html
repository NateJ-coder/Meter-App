QR Codes

How to use

  1. Select a scheme (or leave blank for all) and optionally filter by meter type
  2. QR codes are generated for every matching meter
  3. Click Print Sheet to print all visible QR codes on an A4 sheet (4 per row)
  4. Cut and attach each QR label to the corresponding meter
  5. When a reader scans it with their phone, they go straight to that meter's reading capture form
Note: Readers need an internet connection and a valid login on their phone to submit readings.
  • Select a scheme from the dropdown
  • A QR code will be generated for that scheme's bulk meter
  • For testing: Click "Test Reader Link" button to open in a new tab
  • Download or print the QR code
  • Attach the QR code to the scheme's bulk meter location

    • Workflow: When a reader scans the QR code, they'll capture the bulk meter reading first, then the app will automatically walk them through all submeters across all buildings in the scheme (organized by building and unit number).

    ⚠️ Current Limitation (localStorage skeleton):
    This skeleton uses browser localStorage, so QR codes only work when scanned on the same device and browser where meters were created.

    Current workaround: A reader can scan on a phone, publish the readings there, download the generated reading package, and then import that JSON package on the dashboard device.

    For production: Add a backend (Phase 2) so data is accessible from any device. Then QR codes will work on field readers' phones without manual transfer.

    For now: Use the "Test Reader Link" button to test the workflow on this computer, or use the phone-to-dashboard transfer package workflow.
